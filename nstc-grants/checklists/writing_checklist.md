# 國科會計畫書撰寫檢核清單

> 依章節逐一核對，確保每個部分的品質達到審查標準。建議在初稿完成後、送件前至少使用本清單檢核兩次。

---

## 使用方式

- 建議在完成每一章節的初稿後，立即核對該章節的清單
- 整份計畫書完成後，再做一次完整的整體核對
- 若某項目無法達到，視為「待改善」，記錄在下方備注欄位後集中修改

---

## 第一章：摘要

### 中文摘要

- [ ] 字數在 500 字以內（不超過）
- [ ] 清楚說明研究背景（2-3 句，帶出為何此研究重要）
- [ ] 明確陳述研究問題或知識缺口（1-2 句，具體不空泛）
- [ ] 說明研究方法（2-3 句，點出核心方法即可，不需詳細）
- [ ] 說明預期貢獻（2-3 句，具體而非「本研究有重要貢獻」）
- [ ] 關鍵詞 3-5 個，涵蓋研究主題的核心概念
- [ ] 使用「本研究」、「本計畫」，不使用第一人稱
- [ ] 閱讀摘要後能清楚理解本研究的核心，不需要讀正文

**常見問題**：
- 摘要只是章節標題的堆疊（缺乏實質內容）
- 摘要比正文更難懂（用了太多術語而無解釋）
- 貢獻描述過於宏大（「將徹底改變XX領域」）

---

### 英文摘要

- [ ] 字數在 500 words 以內
- [ ] 內容與中文摘要完全對應（非機器翻譯）
- [ ] 學術英文語法正確，句子結構清晰
- [ ] 已請英語能力較強者（或潤稿服務）校閱
- [ ] Keywords 與中文關鍵詞對應，使用學術界慣用的英文術語

---

## 第二章：研究動機與目的

### 研究背景與重要性

- [ ] 從宏觀（領域）到微觀（具體問題）有清楚的漏斗式結構
- [ ] 引用了近 5 年以內的統計數據或研究成果佐證重要性
- [ ] 說明了「為何此問題現在特別重要」（時代性或緊迫性）
- [ ] 每一段論述都為後續研究問題的出現做了鋪墊

### 三層論證結構

- [ ] **第一層**：現有研究已知道什麼（文獻現況清楚呈現）
- [ ] **第二層**：現有研究還不清楚什麼（知識缺口明確指出）
- [ ] **第三層**：這個缺口為何重要（研究必要性有力論述）

**自我測試**：審查委員讀完「研究問題與知識缺口」後，能否清楚說出「你的計畫要填補什麼缺口」？

- [ ] 知識缺口描述具體（非「目前研究不足」，而是「[具體情境]下[具體方面]尚未有研究探討」）
- [ ] 知識缺口與後續研究目的直接對應

### 研究目的

- [ ] 列出 3-5 個研究目的（不多不少）
- [ ] 每個目的都具體、可判斷達成與否
- [ ] 研究目的直接回應前述知識缺口
- [ ] 每個研究目的在後續「預期成果」中都有對應的成果

**常見問題**：
- 研究目的過於宏大（「建立完整的XX理論框架」）
- 研究目的太籠統（「探討XX與XX的關係」，需說明探討哪個面向）
- 研究目的與預期成果脫節

---

## 第三章：文獻探討

### 組織方式

- [ ] 採主題式組織（不是按發表時間排列）
- [ ] 每個主題子章節有清楚的起承轉合，不是文獻的堆砌
- [ ] 子章節之間有明確的邏輯連結（如：「在了解XX基礎後，進一步探討XX」）

### 文獻品質與比例

- [ ] 近 5 年文獻佔比 **60% 以上**
- [ ] 引用了本領域的頂尖期刊論文（非只有研討會論文）
- [ ] 引用的文獻均有在正文中實質討論（非只是列出引用）
- [ ] 文獻評析有自己的判斷（非只轉述，而是說明其貢獻與限制）

### 比較分析表

- [ ] 包含一個「文獻比較分析表」，列出 5-8 篇最相關的研究
- [ ] 比較維度選取得當（反映研究問題的核心面向）
- [ ] 比較表中有「本計畫」一行，清楚呈現本研究的定位
- [ ] 從比較表可以一眼看出本研究的獨特貢獻

### 文獻小結

- [ ] 有明確的「文獻小結」段落（可獨立為一個子章節）
- [ ] 小結歸納了現有研究的主要成就
- [ ] 小結明確指出本研究如何在既有基礎上推進知識前沿
- [ ] 小結與後續研究方法的選擇有邏輯連結

**常見問題**：
- 文獻探討成了「寫書心得」（只摘要，不評析）
- 忘記近期重要論文（特別是近 1-2 年的新發展）
- 文獻探討與研究方法之間缺乏連結（看不出文獻如何支撐方法選擇）

---

## 第四章：研究方法與步驟

### 研究架構圖

- [ ] 有研究架構圖（或概念框架圖）
- [ ] 架構圖清楚呈現研究問題、方法、預期產出的邏輯關係
- [ ] 圖的品質清晰（非草圖，符合出版品質）
- [ ] 圖後有充分的文字說明（3-5 段）

### 方法選擇理由

- [ ] **說明了為何選擇此方法**（不只是描述方法本身）
- [ ] 比較了替代方法，說明本研究選擇此方法的優勢
- [ ] 方法與研究問題有明確的適配性說明

**自我測試**：若審查委員問「你為何用這個方法而不用[替代方法]？」，你的計畫書有清楚回答嗎？

### 方法細節

- [ ] 研究步驟說明清楚，審查委員可以理解如何執行
- [ ] 若涉及實驗設計，說明了控制變數、樣本大小、統計方法
- [ ] 若涉及計算/模擬，說明了模型假設、驗證方式
- [ ] 若涉及質性研究，說明了訪談對象選取、編碼方式、信效度
- [ ] 技術參數與規格說明有根據（非隨意填寫）

### 工作項目劃分

- [ ] 工作包（WP）劃分合理（3-5 個，不多不少）
- [ ] 每個 WP 的目標、任務、產出都清楚描述
- [ ] WP 之間的邏輯關係明確（先後順序或並行關係）

### 研究時程

- [ ] 時程以甘特圖或表格呈現，一目了然
- [ ] 各工作包的起訖時間合理（總工作量與期程相稱）
- [ ] 論文撰寫/投稿的時間已納入時程（通常後 1/3 時段）
- [ ] 設有清楚的里程碑（Milestone），有具體的判斷標準
- [ ] 時程未過度樂觀（留有緩衝空間）
- [ ] 多年期計畫各年度的工作有明確的分工與遞進關係

### 可行性

- [ ] 說明主持人及團隊有執行本計畫的能力
- [ ] 列出已有的初步研究成果（最重要的加分項）
- [ ] 說明現有設備與資源足以支撐研究執行
- [ ] 若有技術風險，說明了應對策略

**常見問題**：
- 方法描述像教科書（說明方法是什麼，但不說明如何應用於本研究）
- 時程過度樂觀（1年內要發表5篇SCI）
- 沒有說明方法的選擇理由
- 缺乏初步成果的呈現

---

## 第五章：預期完成工作項目及成果

### 學術成果

- [ ] 期刊論文數量合理（1年期：1-2篇；2年期：2-3篇；3年期：3-4篇）
- [ ] 指定的投稿期刊等級與研究貢獻相稱（不過度承諾 Q1 期刊）
- [ ] 研討會論文已列入（通常比期刊先產出，作為中間成果）

### 成果與目的的對應

- [ ] 每個研究目的都有對應的預期成果
- [ ] 製作了「研究目的 vs. 預期成果」對應表
- [ ] 成果描述具體可衡量（非「將有豐碩的研究成果」）

### 應用與社會影響

- [ ] 說明了研究成果的潛在應用方向（即使是基礎研究也應提及）
- [ ] 若有具體應用場景，已說明預期效益（量化較佳）

### 人才培育

- [ ] 列出訓練學生的數量與類型（博士/碩士/大學部）
- [ ] 簡述學生將獲得的研究技能訓練

**常見問題**：
- 預期成果不具體（「本計畫將產生豐碩的成果」）
- 成果與研究目的脫節（預期成果無法回應研究目的）
- 過度承諾（預期發表篇數遠超出合理範圍）

---

## 第六章：經費編列

### 人事費

- [ ] 人事費總額未超過計畫總額的 60%
- [ ] 每位人員的月支金額符合國科會規定標準
- [ ] 每位人員的工作職責與本計畫相關（有說明其負責工作）
- [ ] 人力投入比例合理（博士後/學生的月數與時程一致）

### 業務費

- [ ] 每項業務費都有說明「必要性」（為何需要此費用）
- [ ] 國際差旅費有說明目標研討會及必要性
- [ ] 軟體授權費有說明用途（非泛稱「研究用軟體」）
- [ ] 論文版面費/開放取閱費有估算依據

### 設備費

- [ ] 單件 10 萬元以上的設備已單獨列項
- [ ] 每件設備有說明規格、用途、必要性
- [ ] 若單位已有類似設備，說明為何需要新購

### 整體一致性

- [ ] 人事費 + 業務費 + 設備費的加總等於申請總額（數字無誤）
- [ ] 經費規模與計畫工作量相稱（不過高也不過低）
- [ ] 多年期計畫各年度金額合理分配

**常見問題**：
- 費用只有金額，缺乏必要性說明
- 人事費超過 60% 上限
- 設備費過高但必要性說明不足
- 各年度金額第一年特別高而後續年度很低（顯得前重後輕）

---

## 第七章：主持人資料

### 著作選取

- [ ] 列出的著作確為「近 5 年代表著作」，不是 10 年前的論文
- [ ] 選取的著作與本計畫最相關（非只列最高 IF 的論文）
- [ ] 標示了 IF 值或 JCR 分區（讓審查委員快速了解期刊水準）
- [ ] 標示了通訊作者（*）或第一作者身分
- [ ] 著作引用格式統一（全部用同一種格式）

### 研究計畫紀錄

- [ ] 列出了近 5 年執行的國科會計畫（含計畫編號）
- [ ] 若有順利結案的計畫，已呈現（展現執行力）
- [ ] 計畫數量未超出申請上限

**常見問題**：
- 著作清單未針對本計畫做選擇（直接貼上 CV 上的所有著作）
- 忘記標示通訊作者，讓審查委員誤以為都是掛名作者

## 整體品質檢核

### 邏輯一致性

- [ ] 研究動機 → 研究問題 → 研究方法 → 預期成果，邏輯鏈完整清楚
- [ ] 各章節之間相互呼應，無矛盾之處
- [ ] 研究目的與預期成果一一對應

### 語言品質

- [ ] 全文無錯別字（建議使用文書軟體拼字功能 + 人工閱讀）
- [ ] 專業術語使用正確且前後一致（同一概念用同一詞彙）
- [ ] 句子通順，避免過長的句子（超過 3 行的句子要拆開）
- [ ] 中英文術語混用有一定規律（初次出現時給中英文對照）

### 視覺呈現

- [ ] 有足夠的圖表輔助說明（建議每 2-3 頁有至少一個圖或表）
- [ ] 所有圖表有編號與標題（如：圖1、表2）
- [ ] 圖表品質清晰，文字可辨識
- [ ] 重要數字或結論有適度強調（粗體或框框）

---

## 格式規範確認

- [ ] 總頁數符合公告上限（通常不超過 20-25 頁，請依學門公告確認）
- [ ] 字體大小符合規定（通常本文 12pt，頁面邊界適當）
- [ ] 行距符合規定（通常 1.5 倍或雙倍行距）
- [ ] 每頁有頁碼
- [ ] PDF 輸出後格式正確（特別注意特殊符號、方程式的顯示）
- [ ] 參考文獻格式統一，符合學門規範
- [ ] 上傳的 PDF 檔案大小符合系統限制

---

## 最後審閱建議

撰寫完成後，建議依以下順序做最終審閱：

1. **自己再讀一遍**：從頭到尾完整閱讀，用審查委員的角度評估
2. **找一位不熟悉此領域的同事閱讀**：測試摘要和研究目的是否清楚易懂
3. **找一位同領域的同事閱讀**：測試方法和文獻是否嚴謹
4. **最後確認數字**：確認所有數字（經費、頁數、人員、時程）前後一致
