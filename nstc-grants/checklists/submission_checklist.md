# 國科會計畫送件前檢核清單

> 送件前的最後防線。請在送出計畫前逐項確認，避免因程序疏漏而被退件或影響審查。

---

## 使用說明

- 本清單建議在「預計送件日前 3 天」開始使用
- 所有項目均需確認，不可跳過
- 若有任何項目無法完成，立即聯繫校內研發處尋求協助

---

## 第一部分：計畫書最終確認

### 1.1 內容最終檢查

- [ ] 計畫書已完成全文最終校閱（至少完整閱讀一遍）
- [ ] 無錯別字、無語病
- [ ] 所有 [待填寫] 標記均已填寫完畢（無遺漏）
- [ ] 中英文摘要字數均符合規定（各 500 字/words 以內）
- [ ] 研究目的與預期成果的對應表已完整填寫
- [ ] 參考文獻格式統一，且正文中每條引用均有對應文獻

### 1.2 頁數與格式

- [ ] 計畫書總頁數符合公告上限
  - 確認頁數上限：[待確認：依學門公告填入] 頁
  - 實際頁數：[待填寫] 頁
- [ ] 字體大小符合規定（請確認學門公告要求）
- [ ] 行距符合規定
- [ ] 頁碼正確（每頁有頁碼且連續）
- [ ] 圖表均有編號與標題

### 1.3 PDF 輸出確認

- [ ] 計畫書已轉成 PDF 格式
- [ ] PDF 開啟後格式正常（字體、排版無亂碼）
- [ ] 特殊符號、數學公式顯示正確
- [ ] 圖片清晰度足夠（不模糊、不像素化）
- [ ] PDF 檔案大小符合 iReS 系統上傳限制（通常 10MB 或 20MB 以內）
- [ ] PDF 已命名為適當的檔名（建議：姓名_計畫名稱縮寫_年度.pdf）

---

## 第二部分：iReS 系統填寫確認

<!--
iReS（學術研發服務網）為國科會計畫線上申請系統
網址：https://ilab.nstc.gov.tw/
所有填寫均需在系統截止前完成
-->

### 2.1 基本資料填寫

- [ ] 計畫中文名稱（與計畫書封面一致）
- [ ] 計畫英文名稱（與計畫書封面一致）
- [ ] 計畫類型選擇正確
- [ ] 申請學門/學域選擇正確
- [ ] 計畫期程填寫正確（起訖年月）
- [ ] 申請金額填寫正確（含各年度明細）

### 2.2 主持人資料

- [ ] 主持人基本資料（姓名、服務單位、職稱）核對正確
- [ ] 主持人聯絡方式（電話、Email）填寫正確
- [ ] 主持人學術著作已在系統中更新（依系統要求）
- [ ] 研究人員編號確認無誤

### 2.3 共同主持人/參與人員（若有）

- [ ] 所有共同主持人已加入計畫，並確認其同意
- [ ] 共同主持人的服務單位與職稱填寫正確
- [ ] 研究助理人員資料已填寫

### 2.4 經費表格

- [ ] iReS 系統中的經費表格填寫完畢
- [ ] 各費用項目的金額與計畫書中的經費表一致
- [ ] 各年度金額填寫正確（多年期計畫）
- [ ] 管理費已依機構比率正確計算（通常由系統自動計算）
- [ ] 總計金額核對無誤

### 2.5 計畫書上傳

- [ ] 計畫書 PDF 已成功上傳至系統
- [ ] 上傳後在系統中預覽確認文件正確
- [ ] 系統顯示上傳成功（有時間戳記）

---

## 第三部分：附件文件確認

### 3.1 必要附件

- [ ] 計畫書 PDF（主計畫書）
- [ ] 主持人著作清單（依系統規定格式或獨立附件）

### 3.2 特定情況所需附件

**涉及人體試驗或人體組織樣本**：
- [ ] IRB（人體試驗委員會）核准文件
- [ ] 若申請時尚未取得，說明預計取得時程

**涉及動物實驗**：
- [ ] IACUC（實驗動物照顧及使用委員會）核准文件
- [ ] 若申請時尚未取得，說明預計取得時程

**整合型計畫**：
- [ ] 各子計畫資料已完整填寫
- [ ] 整合計畫書已上傳
- [ ] 各子計畫主持人均已確認參與

**產學合作計畫**：
- [ ] 合作企業同意書（依國科會規定格式，有企業負責人簽章）
- [ ] 企業配合款承諾書
- [ ] 企業基本資料文件

**國際合作計畫**：
- [ ] 海外合作機構確認函
- [ ] 雙邊申請的對應文件（若需要）

## 第四部分：校內行政流程

### 4.1 系所層級

- [ ] 系所辦公室知悉本次申請
- [ ] 系主任/所長已確認（若學校規定需系所層級簽章）
- [ ] 系所行政截止日：[待填寫：YYYY/MM/DD]

### 4.2 院級（若需要）

- [ ] 院辦公室知悉本次申請
- [ ] 院長已確認（若學校規定需院層級簽章）
- [ ] 院行政截止日：[待填寫：YYYY/MM/DD]

### 4.3 研發處層級

- [ ] 已與學校研發處確認本次申請
- [ ] 了解研發處的校內截止日：[待填寫：YYYY/MM/DD]
- [ ] 研發處需要的文件已準備齊全
- [ ] 管理費比率已與研發處確認
- [ ] 研發處已完成系統端的作業（送件由研發處統一辦理）

### 4.4 電子簽章（若適用）

- [ ] 了解學校的電子簽章流程
- [ ] 主持人電子憑證有效且可正常使用
- [ ] 若需主管簽章，已提前安排（勿在截止日當天才請主管簽章）

## 第五部分：截止日期再確認

### 5.1 重要時間節點

| 時間節點 | 日期 | 確認狀態 |
|----------|------|----------|
| 系所行政截止 | [待填寫：YYYY/MM/DD] | [ ] 已確認 |
| 院行政截止 | [待填寫：YYYY/MM/DD 或 N/A] | [ ] 已確認 |
| 研發處收件截止 | [待填寫：YYYY/MM/DD] | [ ] 已確認 |
| 國科會 iReS 系統截止 | [待填寫：YYYY/MM/DD 時間] | [ ] 已確認 |

### 5.2 時間緩衝確認

- [ ] 所有文件已在研發處截止日前 **1 個工作天**完成（預留 buffer）
- [ ] 確認不在截止日當天才送件（系統可能壅塞）
- [ ] 若逼近截止日，已聯繫研發處確認仍可受理

## 第六部分：最終確認清單

### 6.1 三重確認

請完成以下三項最終確認：

**確認一：計畫書完整性**
- [ ] 計畫書從第一頁到最後一頁完整閱讀一遍
- [ ] 所有章節均已完成，無空白頁或 [待填寫] 殘留

**確認二：系統資料一致性**
- [ ] iReS 系統中的計畫名稱、金額、期程與計畫書完全一致
- [ ] 所有必要附件均已上傳

**確認三：行政流程完成**
- [ ] 研發處已確認收到本計畫申請
- [ ] 取得研發處的收件確認（截圖或 Email）

---

## 第七部分：送件後行動

### 7.1 立即行動

- [ ] 記錄 iReS 系統的計畫申請編號：[待填寫]
- [ ] 截圖或列印系統送件成功的確認畫面
- [ ] 備份所有計畫書相關文件（計畫書 PDF、附件、iReS 截圖）

### 7.2 等待期間準備

- [ ] 了解本學門的審查時程（通常 2-4 個月）
- [ ] 若有補件通知，在期限內回覆
- [ ] 準備口試說明（部分計畫類型有口頭審查，如整合型計畫）

### 7.3 審查結果

| 結果 | 行動 |
|------|------|
| **通過（核定補助）** | 確認核定金額、確認執行期程、辦理計畫開始手續 |
| **有條件通過** | 仔細閱讀審查意見，依限回覆修正說明 |
| **未通過** | 申請審查意見書，了解未通過原因，作為下次申請的改進依據 |
