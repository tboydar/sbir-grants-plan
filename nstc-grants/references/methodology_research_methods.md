# 研究方法與步驟撰寫方法論

> ⚠️ **重要聲明**：本方法論基於國科會審查實務經驗，僅供撰寫輔助參考。

---

## 審查委員在研究方法中看什麼？

| 評估項目 | 審查重點 | 常見失敗原因 |
|----------|----------|--------------|
| **方法適當性** | 方法是否能回答研究問題 | 方法與問題不對應 |
| **嚴謹性** | 是否有系統性的實驗設計 | 缺乏控制組、樣本太小 |
| **創新性** | 方法上是否有新穎之處 | 完全沿用前人方法無改進 |
| **可行性** | 在提出的期程內能否完成 | 方法過於理想化 |
| **具體性** | 是否描述得夠清楚可複製 | 太空泛、缺乏細節 |

## 研究架構圖（非常重要）

審查委員通常先看圖再看文字。一張好的研究架構圖能：
- 讓審查委員快速理解整體規劃
- 展示各步驟之間的邏輯關係
- 顯示創新之處在哪裡

### 架構圖要素

```
[輸入/資料] → [方法一] → [中間產出]
                              ↓
                         [方法二] → [最終成果]
                              ↓
                         [驗證/評估]
```

### 架構圖常見問題

| 問題 | 改善方式 |
|------|----------|
| 太複雜看不懂 | 分層呈現，主圖+子圖 |
| 太簡單沒內容 | 加入具體方法名稱與參數 |
| 只有方塊箭頭 | 加入說明文字與數據流 |
| 看不出創新在哪 | 用顏色或標記標示新穎之處 |

## 執行時程表

### 甘特圖要素

| 項目 | 說明 |
|------|------|
| **工作項目** | 每個研究步驟 |
| **時間軸** | 月/季為單位 |
| **里程碑** | 關鍵查核點 |
| **交付物** | 每階段的具體產出 |

### 多年期計畫的時程規劃

| 年度 | 重點 | 關鍵產出 |
|------|------|----------|
| **第一年** | 方法開發 + 初步驗證 | 核心演算法、初步實驗結果、研討會論文 |
| **第二年** | 完整實驗 + 效能優化 | 完整實驗數據、期刊論文投稿 |
| **第三年** | 擴展應用 + 成果整合 | 系統整合、期刊論文發表、技術報告 |

## 方法描述的深度

### 「恰到好處」的原則

- **太淺**：「我們將使用深度學習來處理」→ 審查委員會質疑你到底會不會
- **太深**：列出所有程式碼細節 → 審查委員沒耐心看完
- **恰好**：描述方法原理、關鍵參數設定、選擇此方法的理由

### 範例

❌ **太空泛**：
> 本研究將使用機器學習方法進行分類。

✅ **適當深度**：
> 本研究採用 XGBoost 作為主要分類模型，選擇原因為：(1) 在結構化資料上的表現穩定優於深度學習模型（Grinsztajn et al., 2022）；(2) 具備內建的特徵重要性評估；(3) 支援遺缺值處理。模型超參數將透過 5-fold cross-validation 配合 Bayesian Optimization（Snoek et al., 2012）進行調校，搜尋範圍包括 max_depth (3-10)、learning_rate (0.01-0.3)、n_estimators (100-1000)。

## 自我檢核清單

- [ ] 是否有研究架構圖/流程圖？
- [ ] 每個研究方法是否說明了「為何選擇這個方法」？
- [ ] 實驗設計是否包含比較基準（baseline）？
- [ ] 評估指標是否明確且與研究目的對應？
- [ ] 時程規劃是否切合實際（不是前鬆後緊）？
- [ ] 是否有風險評估與備案？
- [ ] 方法描述是否足夠具體，讓同行能理解執行細節？
- [ ] 多年期計畫是否每年都有明確的查核點？
